# Чеклист: «оживить» интерфейс (вовлечение / вайб)

Передаточный документ для агента. Цель — сделать платформу живой и мотивирующей,
чтобы ученику хотелось возвращаться. Идеи отранжированы по ROI (эффект ÷ труд) и
**приземлены на реальный код** — у каждой указаны файлы, данные и подводные камни.

> **North star.** Тихий дата-инструмент, который в нужную секунду радуется вместе с
> тобой: каждое решение на глазах двигает балл, серию и личную цель — и всегда
> оставляет конкретный повод вернуться завтра. По-взрослому, без детсада (вкус
> платформы из `plan-claude-design.md`: «спокойно, сфокусированно, мотивирующе»).

Обозначения труда: **S** = день-два · **M** = ~неделя · **L** = спринт.
`[x]` — уже сделано в этом проходе, `[~]` — сделано частично (остаток описан), `[ ]` — не начато.

## Правила игры (соблюдать во всех задачах)

- **Токены темы** — только из `web/src/theme.css` (`--accent`, `--accent-2`,
  `--accent-soft`, `--bad`, `--warn`, `--warn-soft`, `--hm0…--hm4`, `--text*`,
  `--surface*`, `--border*`, `--shadow*`). Никаких хардкод-цветов — иначе тёмная
  тема поедет. Цвет по точности брать из `accColor()` (`web/src/ui.tsx`).
- **Анимации** — всегда под `@media (prefers-color-scheme)` нет, а под
  `@media (prefers-reduced-motion: reduce)` **выключать** (блок уже есть в
  `theme.css`, дописывать туда новые классы).
- **Один Modal** — все диалоги через `Modal` из `web/src/ui.tsx` (тема теряется
  при голом `createPortal`, см. коммент в файле).
- **`TaskView` никогда не содержит правильный ответ** — проверка серверная.
- **Схема БД — только через миграции**, по одной на изменение, следующий номер
  **`00010_…`** (последняя — `00009_composed_test_kind.sql`). После миграции:
  правь `internal/store/queries/*.sql` → `make sqlc` → маппинг в `internal/store`.
  **Никогда не редактировать `internal/store/sqlc/` руками.**
- **Новый вид уведомления** = расширить CHECK у `notifications.kind`. Точный
  образец — `migrations/00007_password_reset.sql` (DROP CONSTRAINT → ADD CONSTRAINT
  со списком + CASE на обязательные поля). Текущие виды в `00004_notifications.sql`.
- **Проверка перед коммитом:** `npm --prefix web install && npm --prefix web run build`
  (это `tsc -b && vite build`) + `npm --prefix web test` (vitest); для Go —
  `make test` (сеть безопасности чекера обязана оставаться зелёной).
