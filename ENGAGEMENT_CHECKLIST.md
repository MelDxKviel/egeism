# Чеклист: «оживить» интерфейс (вовлечение / вайб)

Передаточный документ для агента. Цель — сделать платформу живой и мотивирующей,
чтобы ученику хотелось возвращаться. Идеи отранжированы по ROI (эффект ÷ труд) и
**приземлены на реальный код** — у каждой указаны файлы, данные и подводные камни.

> **North star.** Тихий дата-инструмент, который в нужную секунду радуется вместе с
> тобой: каждое решение на глазах двигает балл, серию и личную цель — и всегда
> оставляет конкретный повод вернуться завтра. По-взрослому, без детсада (вкус
> платформы из `plan-claude-design.md`: «спокойно, сфокусированно, мотивирующе»).

Обозначения труда: **S** = день-два · **M** = ~неделя · **L** = спринт.
`[x]` — уже сделано в этом проходе, `[~]` — сделано частично (остаток описан), `[ ]` — не начато.

## Правила игры (соблюдать во всех задачах)

- **Токены темы** — только из `web/src/theme.css` (`--accent`, `--accent-2`,
  `--accent-soft`, `--bad`, `--warn`, `--warn-soft`, `--hm0…--hm4`, `--text*`,
  `--surface*`, `--border*`, `--shadow*`). Никаких хардкод-цветов — иначе тёмная
  тема поедет. Цвет по точности брать из `accColor()` (`web/src/ui.tsx`).
- **Анимации** — всегда под `@media (prefers-color-scheme)` нет, а под
  `@media (prefers-reduced-motion: reduce)` **выключать** (блок уже есть в
  `theme.css`, дописывать туда новые классы).
- **Один Modal** — все диалоги через `Modal` из `web/src/ui.tsx` (тема теряется
  при голом `createPortal`, см. коммент в файле).
- **`TaskView` никогда не содержит правильный ответ** — проверка серверная.
- **Схема БД — только через миграции**, по одной на изменение, следующий номер
  **`00010_…`** (последняя — `00009_composed_test_kind.sql`). После миграции:
  правь `internal/store/queries/*.sql` → `make sqlc` → маппинг в `internal/store`.
  **Никогда не редактировать `internal/store/sqlc/` руками.**
- **Новый вид уведомления** = расширить CHECK у `notifications.kind`. Точный
  образец — `migrations/00007_password_reset.sql` (DROP CONSTRAINT → ADD CONSTRAINT
  со списком + CASE на обязательные поля). Текущие виды в `00004_notifications.sql`.
- **Проверка перед коммитом:** `npm --prefix web install && npm --prefix web run build`
  (это `tsc -b && vite build`) + `npm --prefix web test` (vitest); для Go —
  `make test` (сеть безопасности чекера обязана оставаться зелёной).

## Волна 1 — клиентский делайт (S, без бэкенда, максимум вайба)

### [x] 1. Салют за верный ответ + комбо-счётчик
**Сделано:** блок «Верно!» въезжает классом `celebrate`, галочка пружинит
(`checkpop`), счётчик серии `combo` со «серия ×N» (с ×2); **конфетти-салют** —
новый `web/src/confetti.ts` (один fixed pointer-events:none canvas + rAF,
цвета из токенов темы через `getComputedStyle(.app)`, сам убирается из DOM),
вызывается в `submit()` при `is_correct`, интенсивность растёт со ступенями
серии 3/5/10; **финальный крупный бёрст** на `Results` при `pct === 100`
(+ подпись «— идеально!»). Всё глушится под `prefers-reduced-motion`
(проверка внутри `confettiBurst`).

### [x] 2. Оживший прогноз: каунтап гейджа + «+N к баллу» на Итогах
**Сделано:** `useCountUp` в `charts.tsx` (rAF, 600 мс, easeOutCubic; смена
значения докручивает от текущего; reduced-motion → сразу финал) — число И дуга
`ScoreGauge` едут вместе (дуга перерисовывается от анимированного значения,
цвет по `accColor` живьём проходит рампу). **Дельта на Итогах:** `Solve`
снимает снапшот `useForecast` до сессии (ref на первые данные),
`finishSession` инвалидирует `forecast`/`heatmap`, `Results` показывает пилюлю
`ForecastDelta` «Прогноз 58 → 60 · ▲ +2 первичн.» (рост — `--accent`, ноль —
нейтраль, падение — `--warn`) с подписью «Прогноз — ориентир, не гарантия».

### [x] 3. Живое пламя стрика + вехи 3/7/14/30
**Сделано:** `StreakBadge` принимает `days` (+ `flame-live`, плюрализация);
**вехи** — пламя красится по ступеням `streakColor` (`--warn` с 3 → `--accent`
с 7 → `--hm4` с 14, с 30 усиленный glow), разовое празднование порога на
дашборде (салют из #1 + toast «N дней подряд — так держать!»), порог в
`localStorage` `egeism.streakSeen.<uid>`, сломанная серия перевзводит вехи
(`streakCelebration`, покрыт vitest в `web/src/engage.test.ts`); **под
угрозой** — если сегодня 0, а вчера серия была (`streakAtRisk`), пилюля
warn с «догорающим угольком» (`flame-ember`, тлеющая opacity-анимация) +
микрокопия «Серия догорает — реши одну задачу сегодня». Чистая логика — в
`web/src/engage.ts`.
- [ ] **Фаза 2 (M, опц.):** вынести стрик на сервер (`Europe/Moscow`, `longest`,
  одна недельная «заморозка»); тогда idle-нудж сможет назвать реальное число (см. задачу 8).

### [x] 4. Цель на день (замыкающееся кольцо)
**Сделано:** компонент `Ring` в `charts.tsx` (SVG-кольцо, дуга доезжает
CSS-переходом `.ring-anim`, при reduced-motion статично); карточка «Цель на
день» на дашборде — `N` адаптивный (`dailyGoal`: среднее по активным дням за
две недели, кламп 5..20, свежий аккаунт — 10; vitest), прогресс — сегодняшняя
ячейка heatmap (`todayTotal`), «реши ещё K задач — и цель дня закрыта» /
«выполнено ✓» (кольцо зеленеет; цель — это темп дня, серию продлевает любая
активность — копия их не смешивает), закрытие празднуется один раз в день
(салют + toast, `localStorage` `egeism.goalDone.<uid>`); `finishSession`
инвалидирует heatmap, так что кольцо растёт после каждой сессии.

### [x] 5. Пустые состояния + тултип хитмэпа
**Сделано:** **портальный тултип** хитмэпа вместо нативного `title=`
(`--surface`/`--border`, radius 10, тень `--shadow-2`; «12 июля · 5 решено ·
4 верно», у пустого дня «нет решений»; порталится в `.app`-скоуп, чтобы токены
не терялись — та же ловушка, что у Modal); **иллюстрированные Empty** —
`Empty` принимает `art?` (line-art в манере `icons.tsx`, три новых иконки):
росток — первый визит (карта номеров), телескоп — пусто/ничего не назначено/
банк пуст, медаль — всё решено (очередь ошибок пуста, номер освоен); экраны
`Solve` без заданий получили осмысленные заголовки вместо «Не получилось».

- **Файлы:** `migrations/00010_*`, `internal/store/queries`, `internal/api`,

### [ ] 9-соц. Рейтинг недели класса + позывной/аноним
- **Цель:** недельный рейтинг внутри класса (сброс в понедельник, скользящее окно
  — крон не нужен): топ-5, твоё место и двое соседей над тобой (досягаемый соперник).
- **Данные:** тот же паттерн, что `ClassMastery` (`GROUP BY att.student_id` в
  `internal/store/queries/classes.sql`), но агрегировать по
  `answers.answered_at >= начало недели` (`Europe/Moscow`).
- **Приватность (обязательно):** `users.display_name` (nullable, позывной) +
  `users.leaderboard_visibility` (имя/позывной/аноним), политика класса задаётся
  учителем, дефолт безопасный; резолв одним хелпером на чтении.
- **Файлы:** миграция (колонки в `users`), `classes.sql`, новый read-only
  `GET /classes/{id}/leaderboard?window=week`, рендер (переиспользовать color-grid из `teacher.tsx`).
- **Труд:** M. **Готово когда:** рейтинг показывает место и соседей, приватность соблюдена по умолчанию.

### [ ] 10. Респект (похвала учителя в один тап)
- **Цель:** кнопка «Респект» у учителя → тёплое именное уведомление ученику
  (+ опц. Telegram). Приватно учитель→ученик, поэтому рейтинг/приватность не задеты.
- **Данные:** причину подсказать из `MasterySeries` (крупный недельный скачок),
  только что закрытого слабого места или вехи стрика.
- **Схема:** новый kind `teacher_kudos` (ALTER CHECK, образец `00007`).
- **Файлы:** миграция, `internal/api`, `teacher.tsx` (страница ученика + color-grid),
  `NotificationsBell` (`shell.tsx`), TG-нотифаер/воркер.
- **Труд:** S. **Готово когда:** учитель жмёт «Респект», ученику падает именное уведомление.
